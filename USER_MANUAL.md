# 📖 CSEC ASTU CPD Editorial Archive — User Manual

> Your complete guide to accessing and using the CPD Editorial Archive website.

---

## 🌐 Getting Started

### How to Access the Website

Open any web browser (Chrome, Firefox, Safari, Edge) and navigate to:

**🔗 [https://csec-cpd-editorials.vercel.app](https://csec-cpd-editorials.vercel.app)**

No login or account is required. The website is **completely free and open** to all CSEC ASTU members and the public.

---

## 🏠 The Landing Page

When you open the website, you'll land on the **Home Page**, which is divided into three scrollable sections:

### Section 1: Hero

This is the first screen you see. It features:
- The **CSEC ASTU logo** and the **CPD Division** title
- Two action buttons:
  - **"Start Exploring"** — Scrolls you down to Upcoming Contests
  - **"Past Editorials"** — Jumps directly to the Past Contests section
- A small **"Scroll"** indicator at the bottom to guide you

> [!TIP]
> The page uses **snap scrolling** — a gentle swipe or scroll will smoothly snap to the next section.

---

### Section 2: Upcoming Contests

This section displays contests that **haven't happened yet**.

Each upcoming contest card shows:
- 🟢 A pulsing **"Upcoming"** badge
- 📅 The **scheduled date**
- 📝 The **contest name and description**
- 📆 A **"Remind Me"** button

#### How to Set a Calendar Reminder

1. Find the upcoming contest you're interested in
2. Click the orange **"Remind Me"** button on the card
3. You will be redirected to **Google Calendar** in a new tab
4. The event title, date, and description are pre-filled for you
5. Click **"Save"** in Google Calendar to add it to your schedule

> [!NOTE]
> If there are no upcoming contests, you'll see a friendly message saying *"No Upcoming Contests"* with a prompt to explore past editorials instead.

---

### Section 3: Past Contests (Latest Contests)

This section lists all **completed contests** with their editorials ready to read.

#### Searching for a Contest

At the top of this section, there is a **search bar**:
- Type any keywords separated by spaces
- The search matches **all words independently** across contest names and descriptions
- For example, typing `div 2` will find contests with both "div" and "2" anywhere in the name or description

#### Browsing Contests

- Contests are displayed in a **3-column grid** (or single column on mobile)
- Each card shows:
  - 🏆 **Contest name**
  - 📅 **Date**
  - 📊 **Number of problems** and **difficulty range**
- **Click any contest card** to open its full editorial page

#### Infinite Scroll

- Only **6 contests** are shown initially for fast loading
- **Scroll down** to automatically load more contests
- You can also click the **"Load More"** button to manually load the next batch
- When all contests are loaded, you'll see *"You've reached the end"*

---

## 📝 The Editorial Page

When you click on a contest card, you're taken to the **Editorial Page** for that specific contest.

### Page Layout

The editorial page contains the following sections from top to bottom:

#### 1. Contest Header (Hero)
- **Contest name** in large text
- **Description** of the contest
- **Date** and **difficulty range**
- **"Go to Contest"** button — Opens the original contest link (e.g., on Codeforces) in a new tab

#### 2. Statistics Bar
Three stat cards showing:
- **Problems** — Total number of problems in the contest
- **Difficulty** — The range of difficulties (e.g., "800 - 1600")
- **Participants** — How many people participated

#### 3. Problem Navigation Bar
- A **sticky navigation bar** appears as you scroll
- It shows numbered buttons for each problem (**1, 2, 3**, etc.)
- **Click any number** to instantly jump to that problem

#### 4. Problem Cards

Each problem is displayed as an **expandable card**:

| Element | Description |
|---|---|
| **Problem Number** | The problem ID (e.g., A, B, C or 1, 2, 3) |
| **Title** | The problem's name |
| **Difficulty Badge** | Color-coded: 🟢 Easy, 🟠 Medium, 🔴 Hard, 🟣 Expert |
| **Tags** | Topic tags like `DP`, `Graphs`, `Greedy`, etc. |

---

### How to Read a Problem Editorial

1. **Click on a problem card** to expand it
2. The card will smoothly reveal:
   - **Problem Breakdown** — A detailed explanation of the problem and its key insight
   - **Step-by-Step Approach** — A numbered list walking through the solution logic
   - **Implementation** — The optimal time/space complexity
   - **Code Solution** — Syntax-highlighted source code

3. **Click the card again** to collapse it

> [!TIP]
> The code solution may be hidden behind a **spoiler overlay**. Click **"Reveal Solution"** to see the code. This is intentional — try to solve the problem yourself before looking at the answer!

---

### Disabled / Coming Soon Problems

Some problems may show a **🍳 Coming Soon** badge instead of a difficulty level. This means:
- The editorial for this problem is **still being written**
- The card **cannot be expanded** — clicking it does nothing
- Check back later once the editorial team has published the solution

---

## 📱 Mobile Usage

The website is **fully responsive** and works great on phones and tablets:
- All sections stack vertically on small screens
- The snap-scroll sections work with touch gestures
- The problem navigation bar scrolls horizontally
- Contest cards display in a single column

## 🔗 Sharing Links

You can share direct links to specific contest editorials with your friends and study groups:

**Link format:**
```
https://csec-cpd-editorials.vercel.app/contest/{contest-id}
```

**Example:**
```
https://csec-cpd-editorials.vercel.app/contest/2026-D2-5
```

When shared on **Telegram, Discord, or Twitter**, the link will display a rich preview card with:
- The CSEC ASTU branding
- The site title and description
- A branded preview image

> [!TIP]
> Share editorial links in your CSEC study groups so members can review problems together!

## ❓ FAQ

**Q: Do I need to create an account?**
A: No. The website is completely open — no login required.

**Q: How often is new content added?**
A: Editorials are published after each CPD contest. The website updates automatically from the CMS.

**Q: The page is not loading / showing an error.**
A: Try refreshing the page. If you see a "Sync Error" message, it may be a temporary network issue. Click **"Retry Connection"** to try again.

**Q: Can I use this on my phone?**
A: Absolutely. The website is fully responsive and optimized for mobile.

**Q: A problem says "Coming Soon" — when will the editorial be available?**
A: The editorial team is still working on it. Check back in a day or two.
